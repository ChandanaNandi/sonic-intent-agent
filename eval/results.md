Methodology

The harness runs agent.py --eval-mode for each prompt in prompts.py. The agent invokes the LLM once and writes the structured tool calls as JSON to stdout. No tools are executed; no SONiC changes are applied; no Batfish verification runs. The harness measures only LLM tool-call accuracy.

Argument matching modes: 'exact' requires all expected args present with exact values and no extras. 'subset' requires expected args present and matching; extras allowed. The admin_status prompts use 'subset' because the LLM may pass additional fields or use synonymous string values.
